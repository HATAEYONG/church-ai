# MBO 목표서 — 솔루션화: 목업 우선 + 설정 기반 API 연동

**승인 일시**: 2026-06-18 (PO 승인 완료)

## 최상위 목표
"멘토·설교 등 모든 AI 기능을 키 없이 목업으로 데모할 수 있게 만들고, 설정 화면에서 업체별 API 키를 입력하면 실연동되도록 통일한다 — 여러 업체 공급(화이트라벨) 전제."

## 핵심 원칙
- 목업 우선: 키 0개에서 음성=브라우저 음성 / 멘토=샘플 응답 / 설교=샘플 단계 스트림
- 설정 연동: /settings에서 키 입력 → 실연동. 업체(배포/브라우저)별 독립.
- 일관 패턴: 음성의 "목업 폴백 + 설정 키" 구조를 멘토·설교에 동일 적용.

## 실행 계획
1. 공유 설정 lib(ai-config.ts) + /settings 통합 설정 페이지
2. 멘토 목업+키연동 (요청 헤더로 키 전달)
3. 설교 목업+키연동
4. 레이트리밋(인메모리, 실키 경로) — 초과 429
5. 멘토 품질(max_tokens↑)
6. ESLint 설정 + 키/클라이언트 해석 DRY 공통화

## 키 연동 방식
브라우저 localStorage BYO 키 → 서버 라우트에 x-ai-key 헤더로 전달 → 라우트는 요청키→환경변수→목업 순.

## 완료 기준
- 키 0개 데모 동작(멘토·설교·음성) — 브라우저
- 설정 키 입력 시 실경로 진입(헤더 포함·분기) — 검증(실생성은 유효키 전제)
- 레이트리밋 초과 429
- build 통과·tsc 0·next lint 통과
