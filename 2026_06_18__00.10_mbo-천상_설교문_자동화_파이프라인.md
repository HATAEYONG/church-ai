# MBO 목표서 — 설교문 자동화 파이프라인

**승인 일시**: 2026-06-18 (PO 승인 완료)

## 최상위 목표
"본문을 입력하면 [주제설정 → 구조분석 → 초안작성 → 다각적 비평 → 수정본] 5단계가 순차 스트리밍으로 자동 진행되고, 결과를 마크다운으로 복사·다운로드하는 교역자용 /sermon 페이지를 구현한다."

## 확정 사항(PO)
- 배치: 새 페이지 /sermon (교역자용)
- 실행: 단계별 순차 스트리밍
- 출력: 화면 표시 + 마크다운 복사/다운로드

## 설계
1. src/lib/sermon-stages.ts — 단계 메타(id/title/emoji/desc) + 스트림 마커(클라이언트 안전, 프롬프트 미포함)
2. src/lib/sermon-pipeline.ts — 단계별 system 프롬프트 + buildUser (서버 전용, 새로 설계)
3. src/app/api/sermon/route.ts — 파이프라인 순차 실행, 단계 마커 끼운 단일 스트림
4. src/app/sermon/page.tsx — 본문 입력 → 5단계 패널 실시간 → 최종본 복사/.md 다운로드
5. src/components/NavBar.tsx — 교역자 메뉴에 "📝 설교준비" 추가

- 다각적 비평 4관점: 본문충실/신학건전/회중적용/구조논리
- 모델: claude-opus-4-8 + adaptive thinking + 스트리밍
- 프롬프트는 서버 전용 모듈에만 두어 클라이언트 번들 노출 방지

## 완료·검증 기준
- npm run build / tsc 통과
- /sermon 렌더·본문 입력·실행·5단계 스트리밍·복사/다운로드 동작(브라우저)
- ANTHROPIC_API_KEY 없을 때 503 안전 안내

## 리스크
- 키 미설정 시 실제 생성 불가 → 503 안내, UI 흐름 검증
- 토큰·시간 → 단계별 max_tokens 합리적 설정
- 교리·민감 주제 → "목회자 최종 검토 필수" 프롬프트 명시
